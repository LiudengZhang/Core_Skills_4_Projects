Core Skills for Claude Code Projects 11 skills

Reusable Claude Code skills for project management, task orchestration, session tracking, and resume-driven OSS contribution.

▦
/dashboard
Generate an HTML dashboard showing current task status, blockers, agent components, recent conversations, and project notes.
  • Multi-agent component tracking (each agent writes its own JSON)
  • Task status synced with TaskList
  • Parses conversation history from local JSONL files
  • Integrates project notes grouped by plan
  • Auto-refresh single-page HTML output
⚙
/dispatch
Create and manage task plans for multi-agent execution. Commander/Executor pattern where one terminal creates plans and others execute them.
  • /dispatch create — structured interview to build a plan
  • /dispatch execute — executor picks up and runs a plan
  • Completion checklists (figures, analysis, writing)
  • Context files for shared project knowledge
  • Dashboard integration on plan completion
⏱
/history
Browse and search conversation history from JSONL files. List sessions, summarize conversations, search for keywords across sessions.
  • /history list [n] — recent sessions with previews
  • /history summarize <id> — date range, topics, files modified
  • /history search "keyword" — cross-session search with context
  • Python helper with parse, summary, and search functions
✎
/notes
Manage project notes linked to tasks, plans, and components. Notes stored as JSON files and displayed on the dashboard grouped by plan.
  • /note add — link to tasks, plans, figures, components
  • 4 note types: finding, decision, blocker, question
  • /note show — filter by plan, task, type, or component
  • /note archive — move resolved notes out of active view
  • Python helper with full CRUD and grouped-by-plan queries
★
/oss
Track open-source contributions optimized for resume impact. Dashboard shows skill/layer coverage, gaps to fill, and draft resume bullets.
  • /oss add — add a contribution with resume bullet
  • /oss status — update PR status (planned/in-review/merged)
  • /oss gaps — identify skill and layer gaps to fill next
  • /oss resume — generate resume section from merged PRs
  • Self-contained HTML dashboard with coverage visualization
◉
oss-recon
Map a repository's conventions, toolchain, test patterns, and active maintainers before your first contribution.
  • Extract from CONTRIBUTING.md, pyproject.toml, .pre-commit-config.yaml
  • Read 3 recent merged PRs for style patterns
  • Identify active maintainers and review turnaround
  • scverse-specific notes (towncrier, numpydoc, AnnData tests)
◆
oss-select
Score issues using rubric + resume bullet filter + 2-deep-1-broad portfolio strategy.
  • Draft resume bullet BEFORE choosing issue (negative filter)
  • 4-dimension scoring rubric (core proximity, maintainer signal, scope, story)
  • Portfolio strategy: 2 deep + 1 broad across layers
  • Interview readiness check for each candidate
▶
oss-first-pr
Hybrid first PR strategy: small-but-real contribution that exercises the full pipeline before escalating.
  • Enforces hybrid approach (not trivial, not too ambitious)
  • Full pipeline walkthrough (fork, pre-commit, test, changelog, CI)
  • Prevents #1 mistake: missing towncrier fragment
  • Success criteria before escalating to substantive work
▣
oss-fix
Implement fixes with self-critique checklist. Claude handles 60-80% of conventions; you handle editorial judgment.
  • Clear split: what Claude automates vs what needs human judgment
  • 7-item pre-submission checklist
  • Adversarial self-review before pushing
  • Scverse-specific pitfall prevention
▲
oss-submit
PR description craftsmanship with dual framing for bioinformatics vs software engineering audiences.
  • Convention-matched PR descriptions
  • Dual framing: domain language vs engineering language
  • Dashboard status update on submission
  • Post-merge interview prep checklist
♡
oss-review
Handle reviewer feedback as a public artifact. Extract interview material from the review process.
  • Response principles: fast, concise, grateful, never argue
  • Common feedback pattern responses
  • Interview material extraction from review threads
  • Graceful handling of PR rejection

Installation

git clone https://github.com/LiudengZhang/Core_Skills_4_Projects.git
cp -r Core_Skills_4_Projects/core_dashboard  your-project/.claude/skills/dashboard
cp -r Core_Skills_4_Projects/core_dispatch   your-project/.claude/skills/dispatch
cp -r Core_Skills_4_Projects/core_history    your-project/.claude/skills/history
cp -r Core_Skills_4_Projects/core_notes      your-project/.claude/skills/notes
cp -r Core_Skills_4_Projects/oss_contributions your-project/.claude/skills/oss
cp -r Core_Skills_4_Projects/oss_recon        your-project/.claude/skills/oss-recon
cp -r Core_Skills_4_Projects/oss_select       your-project/.claude/skills/oss-select
cp -r Core_Skills_4_Projects/oss_first_pr     your-project/.claude/skills/oss-first-pr

Each skill folder contains a SKILL.md file (and optional helpers) that Claude Code loads automatically from .claude/skills/.
